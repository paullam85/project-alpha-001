# Reddit Case Scoring Rules

This scoring guide keeps manually curated Reddit cases consistent and useful for Roof AI product decisions.

## Score Definitions

| Score | Meaning | Use When |
| --- | --- | --- |
| A | Strong product signal | The case shows clear homeowner decision uncertainty, repeated pain, strong comments, or a direct product module opportunity. |
| B | Useful supporting evidence | The case supports an existing pattern but has less urgency, fewer comments, or weaker monetization signal. |
| C | Low-value technical discussion | The case is mostly narrow technical detail, isolated opinion, or not clearly tied to a homeowner decision. |

## Priority Criteria

Raise priority when a case includes one or more of these signals:

- High comment count or active discussion
- Repeated homeowner pain
- Decision uncertainty
- Insurance impact
- Contractor trust issue
- Quote comparison issue
- Attic heat, ventilation, or insulation issue
- Clear possible product feature

## First Theme Categories

- Attic Heat
- Ventilation Confusion
- Insurance Anxiety
- Contractor Trust
- Quote Comparison
- Repair vs Replace
- Renovation Sequencing
- Roof Age Anxiety

## Validation Signal Framework

| Signal | Meaning |
| --- | --- |
| Red | Low evidence of urgency or willingness to act. |
| Yellow | Some evidence of concern, but unclear intent or weak conversion path. |
| Green | Strong evidence of a painful decision, active information seeking, and possible product fit. |

## Weekly Review Questions

- Which cases repeated the same homeowner anxiety?
- Which objections appeared more than once?
- Which product modules map to the highest-priority cases?
- Which cases suggest a lead-generation angle?
- Which findings should change the landing page, report logic, or future feature roadmap?
